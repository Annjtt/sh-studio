# Руководство по подключению к GitHub репозиторию

## 1. Подготовка к работе

### Установка Git
1. Скачайте Git с официального сайта: https://git-scm.com/downloads
2. Установите Git, следуя инструкциям установщика
3. Откройте терминал и проверьте установку командой: git --version

### Настройка Git
1. Настройте ваше имя пользователя:
   git config --global user.name "Ваше Имя"
2. Настройте email:
   git config --global user.email "ваш@email.com"

## 2. Подключение к GitHub

### Создание SSH ключа
1. Откройте терминал
2. Выполните команду:
   ssh-keygen -t ed25519 -C "ваш@email.com"
3. Нажимайте Enter для принятия значений по умолчанию
4. Найдите созданный ключ в директории ~/.ssh/

### Добавление SSH ключа в GitHub
1. Скопируйте содержимое файла ~/.ssh/id_ed25519.pub
2. Зайдите на GitHub.com и войдите в свой аккаунт
3. Перейдите в Settings -> SSH and GPG keys
4. Нажмите New SSH key
5. Вставьте скопированный ключ и сохраните

## 3. Подключение к репозиторию

### Создание нового репозитория
1. На GitHub нажмите + -> New repository
2. Заполните название и описание
3. Выберите настройки приватности
4. Создайте репозиторий

### Подключение существующего проекта
1. Перейдите в директорию проекта:
   cd путь/к/проекту
2. Инициализируйте Git:
   git init
3. Добавьте файлы в репозиторий:
   git add .
4. Создайте первый коммит:
   git commit -m "Initial commit"
5. Подключите удаленный репозиторий:
   git remote add origin git@github.com:username/repository.git
6. Отправьте изменения:
   git push -u origin main

### Клонирование существующего репозитория
1. Найдите URL репозитория на GitHub
2. Выполните команду:
   git clone git@github.com:username/repository.git

## 4. Проверка подключения

1. Проверьте статус:
   git status
2. Создайте тестовый файл и отправьте его:
   echo "# Test" > test.md
   git add test.md
   git commit -m "Test commit"
   git push

## 5. Решение проблем

### Ошибка аутентификации
1. Проверьте наличие SSH ключа:
   ls -la ~/.ssh
2. Проверьте подключение к GitHub:
   ssh -T git@github.com

### Ошибка push/pull
1. Убедитесь, что вы в правильной ветке:
   git branch
2. Проверьте remote URL:
   git remote -v
3. Попробуйте получить последние изменения:
   git pull origin main

### Конфликты слияния
1. Получите последние изменения:
   git pull origin main
2. Решите конфликты в файлах
3. Добавьте решенные файлы:
   git add .
4. Создайте коммит слияния:
   git commit -m "Resolve merge conflicts"
5. Отправьте изменения:
   git push origin main
